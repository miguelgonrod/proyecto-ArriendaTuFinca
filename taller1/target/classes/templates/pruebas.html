Pruebas

Casos de pruebas frontend

  • Pruebas de Interfaz de Usuario (UI)
    • Verificación de la Navegación: Asegúrate de que todos los enlaces de navegación funcionan correctamente para ambos perfiles de usuario (Arrendatario y Arrendador).
    • Prueba de Formularios de Propiedad: Verifica que los formularios para la creación, edición, y eliminación de propiedades por parte del "Usuario Arrendatario" funcionen correctamente.
    • Prueba de Filtrado de Propiedades: Asegúrate de que el "Usuario Arrendador" pueda filtrar las propiedades por ubicación geográfica de manera efectiva.
    • Validación de Campos de Formulario: Comprueba que todos los campos de los formularios tengan las validaciones adecuadas (ej., campos obligatorios, formato de datos correcto).
    • Prueba de Respuesta a Solicitudes: Verifica que el "Usuario Arrendatario" pueda ver y responder a las solicitudes de arrendamiento de manera eficiente.
    • Prueba de Interfaz de Pago: Asegúrate de que el proceso de pago por parte del "Usuario Arrendador" funcione correctamente y sea intuitivo.
    • Prueba de Calificaciones y Comentarios: Verifica que ambos tipos de usuarios puedan dejar calificaciones y comentarios sobre sus experiencias.
  • Pruebas de Responsividad
    • Compatibilidad con Móviles: Asegúrate de que la interfaz sea completamente funcional y estéticamente agradable en dispositivos móviles.
    • Compatibilidad con Diferentes Navegadores: Prueba la aplicación en múltiples navegadores (Chrome, Firefox, Safari, Edge) para garantizar la compatibilidad.
  • Pruebas de Accesibilidad
    • Prueba de Teclado: Verifica que todas las funcionalidades sean accesibles mediante el teclado.
    • Prueba de Lectores de Pantalla: Asegúrate de que la aplicación sea compatible con lectores de pantalla para usuarios con discapacidades visuales.

Casos de pruebas backend

  • Pruebas de API
    • Creación de Propiedades: Asegúrate de que la API permite al "Usuario Arrendatario" crear nuevas propiedades correctamente.
    • Edición de Propiedades: Verifica que las propiedades se puedan actualizar con los datos correctos.
    • Eliminación de Propiedades: Asegúrate de que las propiedades se eliminen correctamente y que se realicen las verificaciones necesarias antes de la eliminación.
    • Solicitud de Arrendamiento: Verifica que el "Usuario Arrendador" pueda enviar solicitudes de arrendamiento y que estas se almacenen correctamente en la base de datos.
    • Gestión de Pagos: Asegúrate de que el sistema maneje correctamente los pagos realizados por el "Usuario Arrendador" y registre estos pagos en la base de datos.
    • Calificaciones y Comentarios: Verifica que ambos tipos de usuarios puedan enviar y recuperar calificaciones y comentarios.
  • Pruebas de Seguridad
    • Autenticación y Autorización: Asegúrate de que solo los usuarios autenticados puedan acceder a las funcionalidades específicas de su rol (Arrendatario o Arrendador).
    • Protección contra SQL Injection: Realiza pruebas para verificar que las entradas de usuario no puedan explotar vulnerabilidades de SQL Injection.
    • Prueba de Sesiones y Tokens: Asegúrate de que las sesiones y tokens de autenticación sean manejados de manera segura y expiren correctamente.
  • Pruebas de Integración
    • Integración de API de Pago: Verifica que la integración con cualquier sistema de pago externo funcione correctamente y maneje adecuadamente las respuestas de error y éxito.
    • Notificaciones y Mensajería: Asegúrate de que las notificaciones se envíen correctamente cuando se aceptan, rechazan o crean solicitudes de arrendamiento.
  • Pruebas de Rendimiento
    • Carga del Sistema: Verifica cómo el sistema maneja múltiples solicitudes de arrendamiento simultáneas.
    • Pruebas de Escalabilidad: Asegúrate de que la aplicación puede escalar adecuadamente para manejar un gran número de propiedades y usuarios.
