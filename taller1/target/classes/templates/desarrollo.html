Endpoints para la Autenticación y Autorización

  • POST /auth/register
    • Descripción: Registra un nuevo usuario (arrendador o arrendatario).
    • Request: Datos del usuario (nombre, correo electrónico, contraseña, rol).
    • Response: Confirmación del registro o error.
  • POST /auth/login
    • Descripción: Autentica al usuario y genera un token JWT.
    • Request: Credenciales del usuario (correo electrónico y contraseña).
    • Response: Token JWT y detalles del usuario.
  • GET /auth/me
    • Descripción: Devuelve la información del usuario autenticado.
    • Request: Token JWT en los headers.
    • Response: Datos del usuario.

Endpoints para la Gestión de Propiedades

  • POST /properties
    • Descripción: Crea una nueva propiedad para arrendar.
    • Request: Datos de la propiedad (nombre, dirección, descripción, precio, etc.).
    • Response: Detalles de la propiedad creada o error.
  • GET /properties
    • Descripción: Obtiene una lista de todas las propiedades disponibles.
    • Request: Opcionalmente, filtros como ubicación, precio, tipo de propiedad, etc.
    • Response: Lista de propiedades.
  • GET /properties/{id}
    • Descripción: Obtiene los detalles de una propiedad específica.
    • Request: ID de la propiedad.
    • Response: Detalles de la propiedad.
  • PUT /properties/{id}
    • Descripción: Actualiza los detalles de una propiedad existente.
    • Request: Datos actualizados de la propiedad.
    • Response: Detalles de la propiedad actualizada o error.
  • DELETE /properties/{id}
    • Descripción: Elimina una propiedad específica.
    • Request: ID de la propiedad.
    • Response: Confirmación de eliminación o error.

Endpoints para la Gestión de Arrendamientos

  • POST /rentals
    • Descripción: Crea una solicitud de arrendamiento para una propiedad.
    • Request: Datos de la solicitud (ID de la propiedad, fechas de arrendamiento, etc.).
    • Response: Detalles de la solicitud creada o error.
  • GET /rentals
    • Descripción: Obtiene una lista de todas las solicitudes de arrendamiento hechas por el usuario autenticado.
    • Request: Token JWT en los headers.
    • Response: Lista de solicitudes de arrendamiento.
  • GET /rentals/{id}
    • Descripción: Obtiene los detalles de una solicitud de arrendamiento específica.
    • Request: ID de la solicitud.
    • Response: Detalles de la solicitud.
  • PUT /rentals/{id}
    • Descripción: Actualiza el estado de una solicitud de arrendamiento (aceptar o rechazar).
    • Request: Nuevo estado de la solicitud.
    • Response: Detalles de la solicitud actualizada o error.

Endpoints para la Gestión de Pagos

  • POST /payments
    • Descripción: Registra un nuevo pago para una solicitud de arrendamiento aceptada.
    • Request: Datos del pago (ID de la solicitud, monto, método de pago, etc.).
    • Response: Confirmación del pago o error.
  • GET /payments
    • Descripción: Obtiene una lista de todos los pagos realizados por el usuario autenticado.
    • Request: Token JWT en los headers.
    • Response: Lista de pagos.
  • GET /payments/{id}
    • Descripción: Obtiene los detalles de un pago específico.
    • Request: ID del pago.
    • Response: Detalles del pago.

Endpoints para la Gestión de Calificaciones y Comentarios

  • POST /reviews
    • Descripción: Crea una nueva calificación y comentario para una propiedad o usuario.
    • Request: Datos de la calificación (ID de la propiedad o usuario, puntuación, comentario).
    • Response: Detalles de la calificación creada o error.
  • GET /reviews
    • Descripción: Obtiene una lista de todas las calificaciones y comentarios hechos por el usuario autenticado.
    • Request: Token JWT en los headers.
    • Response: Lista de calificaciones.
  • GET /reviews/{id}
    • Descripción: Obtiene los detalles de una calificación específica.
    • Request: ID de la calificación.
    • Response: Detalles de la calificación.

Endpoints para la Gestión de Usuarios

  • GET /users

    • Request: Token JWT en los headers.
    • Response: Lista de usuarios.
  • GET /users/{id}
    • Descripción: Obtiene los detalles de un usuario específico.
    • Request: ID del usuario.
    • Response: Detalles del usuario.
  • PUT /users/{id}
    • Descripción: Actualiza los detalles de un usuario (solo para administradores).
    • Request: Datos actualizados del usuario.
    • Response: Detalles del usuario actualizado o error.
  • DELETE /users/{id}
    • Descripción: Elimina un usuario específico (solo para administradores).
    • Request: ID del usuario.
    • Response: Confirmación de eliminación o error.
